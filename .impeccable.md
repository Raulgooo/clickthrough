## Design Context

### Users

**Primary**: Hackathon judges evaluating a 2-minute Generative UI demo. They need to immediately understand that Clickthrough is NOT a chatbot and that the UI is generated at runtime for different intents.

**Secondary**: End-users who would install Clickthrough as a browser layer. They need to feel confidence, speed, and intelligence when an overlay appears over their current page.

**Context**: The product is invoked in-place on any webpage. The user is usually in the middle of a task (scrolling Twitter, reading a PDF, using a dashboard, messaging). The overlay must feel like it belongs to the page while remaining recognizably Clickthrough-controlled.

### Brand Personality

**Three words**: Precise. Dense. Engineered.

**Voice**: Hyper-competent infrastructure. Like a forensic tool or IDE that assembled itself because it understood your intent instantly. No decoration, no warmth without purpose. The interface speaks in data, status, and clear hierarchies.

**Emotional goals**:
- Confidence: "This thing knows exactly what it's doing."
- Speed: "It appeared before I finished thinking."
- Authority: "Every pixel has a job."

### Aesthetic Direction

**Visual tone**: Brutalist utility. High-density monochrome interface. Material Design 3 structural roles executed in pure grayscale. Hard shadows for emphasis. Tiny, intentional typography. The interface feels cut from paper, not rendered from gradients.

**References**:
- **IDE / Developer Tools**: JetBrains Mono labels, dense tool panels, status indicators, hard edges
- **Material Design 3**: Surface/container hierarchy, role-based color tokens, elevation through tone
- **Raycast**: Command-speed invocation, compact UI

**Anti-references**:
- ChatGPT sidebar (explicitly NOT a chat interface)
- Generic SaaS dashboard (no hero metrics, no gradient cards)
- Glassmorphism-heavy interfaces
- Purple-blue gradient "AI" aesthetic
- Soft, rounded, "friendly" consumer UI

**Theme**: Light mode by default with full dark mode support (`darkMode: "class"`). The base palette is pure grayscale with black primary. Host page adaptation shifts accent hue while the CT skeleton remains monochrome.

### Design Principles

1. **Every pixel is functional**: 2px spacing unit (`xs`), 11px labels, 1px borders (`border-1`). No decorative padding. Density is the default.

2. **Hard edges, hard shadows**: Flat surfaces with `2px 2px 0px #000` hard shadows create depth without blur. Emphasis is physical, not atmospheric.

3. **Monochrome Material architecture**: Use MD3 surface/container/role hierarchy (`surface-container-lowest` through `surface-container-highest`, `primary-container`, `secondary-container`, `tertiary-container`) but without hue. Elevation is communicated through gray values.

4. **Type is infrastructure**: JetBrains Mono for all metadata, labels, status, and tabular data. Geist for body and headlines. No display fonts — the interface IS the content. Sizes stay between 11px and 16px.

5. **The overlay is the product**: There is no separate "app." The generated overlay IS Clickthrough. The design system exists to make runtime-generated UIs feel intentional, safe, and fast.

### Font Choices

- **Body / Headlines / UI**: Geist (400, 600) — geometric, precise, fast-feeling at small sizes. Excellent for dense interfaces.
- **Labels / Mono / Tabular / Status**: JetBrains Mono (400, 500) — authoritative, dense, excellent for metadata and system state.

Both loaded from Google Fonts. System fonts as fallbacks.

### Color Philosophy

- **Role-based monochrome**: Material Design 3 token architecture (`surface`, `on-surface`, `primary`, `on-primary`, `outline`, `error`, `surface-container-*`, etc.) adapted to pure grayscale.
- **Primary is pure black** (`#000000`) in light mode. Surfaces range from white (`#ffffff`) through `#f9f9f9`, `#eeeeee`, `#e2e2e2`, `#dadada`.
- **No gradients**. No tinted neutrals. Pure grayscale for maximum host-page adaptability.
- **Error red** (`#ba1a1a`) is the only non-gray color in the base system.
- **Host adaptation**: When blending into a host page, only accent hue shifts. The CT overlay skeleton remains monochrome and recognizable.
- Dark mode mirrors the same roles with inverted values via `darkMode: "class"`.

### Spacing & Shape

- **Spacing scale**: `xs: 2px`, `sm: 4px`, `md: 8px`, `lg: 12px`, `xl: 16px`. `margin-edge: 12px`, `gutter: 8px`.
- **Border radius**: Very tight. `DEFAULT: 0.25rem`, `lg: 0.5rem`, `xl: 0.75rem`. No excessive rounding.
- **Shadows**: `hard-shadow: 2px 2px 0px rgba(0,0,0,1)` for emphasis. `subtle-lift: 1px 1px 3px rgba(0,0,0,0.1)` for minor elevation.
- **Borders**: 1px solid `outline` or `outline-variant` for structure.

### Motion

- Transitions are fast and functional (`duration-150`, `ease-in-out`).
- No bounce, elastic, or decorative animation.
- Respect `prefers-reduced-motion`.
- Loading states feel like assembly, not waiting.
